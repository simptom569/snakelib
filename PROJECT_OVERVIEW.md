# SnakeLab — Обзор проекта

> Этот документ написан простым языком. Здесь нет сложных технических терминов — только суть того, что представляет собой проект, из чего состоит и как работает.

---

## Что такое SnakeLab?

**SnakeLab** — это интернет-магазин для любителей рептилий. На сайте можно:

- Купить **змей** разных видов и окрасов
- Купить **террариумы** для содержания рептилий
- Купить **корм** для змей
- Зарегистрироваться, войти в аккаунт и отслеживать свои заказы
- Управлять адресами доставки
- Применять промокоды со скидками

Проект представляет собой полноценный интернет-магазин с покупательской частью и панелью администратора.

---

## Из чего состоит проект

Проект разделён на три части, которые работают вместе:

### 1. Сайт (Frontend — то, что видит пользователь)

Это визуальная часть сайта — страницы, кнопки, карточки товаров. Пользователь открывает браузер и видит именно эту часть.

**Страницы сайта:**

| Страница | Адрес | Что делает |
|---|---|---|
| Главная | `/` | Приветственная страница с описанием магазина и акциями |
| Каталог змей | `/catalog` | Список всех змей с фильтрами |
| Страница змеи | `/catalog/название` | Подробная информация о конкретной змее |
| Каталог террариумов | `/terrariums` | Список всех террариумов |
| Страница террариума | `/terrariums/название` | Подробная информация о террариуме |
| Каталог корма | `/food` | Список всех кормов |
| Страница корма | `/food/название` | Подробная информация о корме |
| Корзина | `/cart` | Добавленные товары, применение промокода, оформление заказа |
| Профиль | `/profile` | Личные данные пользователя, адреса, смена пароля |
| Заказы | `/orders` | История заказов и их статусы |
| Вход / Регистрация | `/login` | Форма входа и регистрации |
| О нас | `/about` | Информация о компании |
| Доставка | `/delivery` | Информация о доставке и оплате |
| FAQ | `/faq` | Часто задаваемые вопросы |
| Панель администратора | `/admin-panel` | Управление магазином (только для администраторов) |

---

### 2. Сервер (Backend — "мозг" сайта)

Это невидимая часть, которая обрабатывает все данные. Когда пользователь нажимает «Добавить в корзину», именно сервер сохраняет эту информацию. Сервер написан на языке программирования **Python**.

---

### 3. База данных

Это "склад" всей информации: товары, пользователи, заказы, промокоды. Все данные хранятся здесь.

---

## Что хранится в базе данных

### Пользователи
- Имя, фамилия, email, телефон
- Пароль (зашифрованный)
- Роль: обычный пользователь или администратор
- Флаг подтверждения email

### Адреса доставки
- Каждый пользователь может сохранить несколько адресов
- Один адрес можно пометить как основной (по умолчанию)

### Змеи
Каждая змея содержит:
- Название, описание, цена, количество в наличии
- Минимальный и максимальный размер (в сантиметрах)
- Диапазон температуры содержания (°C)
- Уровень сложности ухода
- Артикул (SKU)
- Изображения
- **Морфы** — это цветовые вариации одной и той же змеи. У каждого морфа может быть своя цена и количество. Например, змея "Корнеус" может иметь морфы: обычный (1000₽), альбинос (+500₽), огненный (+2000₽)
- Характеристики: продолжительность жизни, влажность, тип кормления, страна происхождения

### Категории змей
- Название, описание, slug (короткое имя для адреса в браузере)
- Можно прикрепить тег (например, "Новинка", "Хит продаж")

### Террариумы
Каждый террариум содержит:
- Название, описание, цена, количество
- Размеры: длина × ширина × высота (в сантиметрах)
- Материал изготовления
- Изображения, артикул

### Корм
Каждая позиция корма содержит:
- Название, описание, цена, количество
- Вес (в граммах)
- Тип животного (мышь, крыса, и т.д.)
- Размер: XS, S, M, L, XL
- Заморожен или живой (флаг is_frozen)
- Изображения, артикул

### Теги
Общие пометки для товаров: "Хит продаж", "Новинка", "Редкость", "Мало в наличии" — каждый тег имеет свой цвет.

### Корзина
- У каждого пользователя есть своя корзина
- В корзине хранятся позиции: какой товар, какой морф (для змей), количество

### Промокоды
- Уникальный код
- Тип скидки: процент от суммы или фиксированная сумма
- Минимальная сумма заказа для применения
- Максимальное количество использований
- Даты действия промокода

### Заказы
Каждый заказ содержит:
- Пользователь, который сделал заказ
- Список товаров с ценами на момент заказа
- Статус: Ожидает → Подтверждён → Оплачен → Отправлен → Доставлен (или Отменён)
- Способ оплаты: карта или наличные
- Итоговая сумма
- Адрес доставки (сохраняется навсегда, даже если пользователь потом удалит адрес)
- Применённый промокод
- Комментарий к заказу

## Как пользователь взаимодействует с сайтом

### Регистрация и вход
1. Пользователь открывает страницу `/login`
2. Заполняет форму: имя, фамилия, email, телефон, пароль
3. Или входит через Google (кнопка "Войти с Google")
4. После входа появляется доступ к корзине, профилю, заказам

### Покупка товара
1. Переходит в каталог `/catalog` (или `/terrariums`, `/food`)
2. Фильтрует по категории, цене, сложности ухода и т.д.
3. Нажимает на карточку товара → видит подробное описание
4. Для змей — выбирает морф (цветовую вариацию)
5. Нажимает "Добавить в корзину"
6. Переходит в корзину `/cart`
7. Вводит промокод (если есть)
8. Нажимает "Оформить заказ"
9. Выбирает адрес доставки, способ оплаты, оставляет комментарий
10. Подтверждает заказ

### Отслеживание заказов
- Страница `/orders` показывает все заказы с их статусами
- Можно видеть: какие товары были куплены, по какой цене, куда доставить, статус

## Панель администратора

Только пользователи с ролью **администратор** имеют доступ к `/admin-panel`.

### Что может делать администратор:

**Дашборд (главная страница)**
- Общее количество заказов
- Заказов сегодня
- Общая выручка
- Заказов в ожидании
- Всего пользователей
- Активных промокодов

**Управление заказами**
- Просмотр всех заказов
- Изменение статуса (например, "Подтверждён" → "Отправлен")
- Фильтрация по статусу и способу оплаты

**Управление пользователями**
- Просмотр всех зарегистрированных пользователей
- Изменение роли (обычный пользователь / администратор)
- Блокировка аккаунта

**Управление товарами**
- Змеи: добавление, редактирование, удаление, управление морфами и изображениями
- Террариумы: добавление, редактирование, удаление, изображения
- Корм: добавление, редактирование, удаление, изображения

**Управление категориями**
- Отдельные разделы для категорий змей, террариумов, корма

**Управление промокодами**
- Создание новых промокодов
- Настройка скидок, сроков, лимитов использования
- Просмотр статистики использования

## Технологии (кратко и понятно)

| Что | Технология | Простое объяснение |
|---|---|---|
| Визуальная часть сайта | React | Библиотека для создания интерактивных страниц |
| Серверная часть | Python / Django | Язык программирования и инструмент для создания веб-серверов |
| База данных | PostgreSQL | Программа для хранения и управления данными |
| Авторизация | JWT-токены | Система "ключей" — после входа выдаётся ключ, подтверждающий личность |
| Авторизация через Google | Google OAuth | Стандартный протокол входа через аккаунт Google |
| Оформление сайта | Styled Components | Способ писать стили прямо в коде страниц |
| Запуск проекта | Docker | Инструмент упаковки всего проекта в контейнеры для удобного запуска |
| Прокси-сервер | Nginx | "Диспетчер" — распределяет запросы между сервером и сайтом |

## Безопасность

- Пароли пользователей **никогда не хранятся в открытом виде** — только зашифрованными
- После входа выдаётся **токен** (временный ключ), который действует 60 минут
- Для долгосрочного входа используется **refresh-токен** на 90 дней
- Токены хранятся в **httpOnly cookie** — JavaScript-код на странице не может их прочитать, это защита от взлома
- Промокоды проверяются на стороне сервера: нельзя применить просроченный или исчерпанный промокод

## Архитектура (как всё связано)

```
Пользователь (браузер)
        ↓
     Nginx (порт 80)
    /          \
   /            \
React           Django
(frontend)      (backend)
порт 3000       порт 8000
                    ↓
               PostgreSQL
               (база данных)
```

**Как это работает:**
1. Пользователь открывает сайт в браузере
2. Запрос приходит на Nginx — это "охранник на входе"
3. Если запрос за данными (API) — Nginx перенаправляет на Django
4. Если запрос за страницей — Nginx перенаправляет на React
5. Django при необходимости обращается к базе данных PostgreSQL
6. Данные возвращаются обратно пользователю

## Возможные вопросы и ответы

**Q: Что такое морфы у змей?**
A: Морф — это цветовая вариация (мутация) одной и той же породы змеи. Например, королевский питон может быть обычным, альбиносом, пегим и т.д. Каждый морф может стоить по-разному. На сайте пользователь выбирает морф при добавлении змеи в корзину.

**Q: Как работает промокод?**
A: Пользователь вводит код в корзине. Сервер проверяет: существует ли такой код, не истёк ли срок действия, не превышен ли лимит использований, не меньше ли сумма заказа минимально допустимой. Если всё хорошо — скидка применяется. Скидка может быть в процентах (например, 10%) или фиксированной суммой (например, 500₽).

**Q: Как сохраняется адрес доставки в заказе?**
A: При оформлении заказа адрес "фотографируется" и сохраняется прямо в заказе. Это значит, что если пользователь потом изменит или удалит адрес, заказ всё равно будет показывать правильный адрес доставки.

**Q: Что происходит, когда токен авторизации истекает?**
A: Токен авторизации живёт 60 минут. Когда он истекает, сайт автоматически (незаметно для пользователя) запрашивает новый токен, используя долгосрочный refresh-токен. Пользователь не замечает этого и не выбрасывается на страницу входа.

**Q: Может ли обычный пользователь зайти в панель администратора?**
A: Нет. Защита работает на двух уровнях: во-первых, сайт не откроет страницу `/admin-panel` для пользователя без роли администратора. Во-вторых, даже если попытаться напрямую обратиться к серверу — сервер проверит роль и откажет в доступе.

**Q: Как работает вход через Google?**
A: Пользователь нажимает "Войти с Google", открывается стандартное окно Google, пользователь разрешает доступ. Google передаёт на сервер подтверждение личности (токен), сервер его проверяет и либо создаёт новый аккаунт, либо авторизует существующий.

**Q: Может ли магазин работать без Docker?**
A: Теоретически да, но Docker используется для удобства запуска. Он упаковывает все части проекта (сайт, сервер, база данных, прокси) и запускает их вместе одной командой. Без Docker каждую часть нужно было бы настраивать отдельно.

**Q: Где хранятся картинки товаров?**
A: Изображения загружаются на сервер и хранятся в файловой системе. В базе данных хранится только путь к файлу. Nginx отдаёт статические файлы напрямую, не нагружая Django.

**Q: Что такое slug?**
A: Slug — это читаемая часть адреса страницы. Например, для змеи "Корнеус" slug может быть `kornevs-uzkaya-zmea`. Благодаря slug адреса страниц выглядят понятно для людей и поисковых систем.
